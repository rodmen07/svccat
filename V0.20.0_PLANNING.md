# svccat v0.20.0 Planning

**Status:** Feature collection phase (v0.19.0 released 2026-05-28)

## Candidate Features

Features listed by estimated value and community demand. Gathering feedback via:
- GitHub Discussions: https://github.com/rodmendoza07/svccat/discussions
- Issue templates: [feature_request.md](.github/ISSUE_TEMPLATE/feature_request.md)

### High-Value Candidates

#### 1. Multi-Repo Support
**Problem:** Users managing multiple git repositories need to run svccat separately for each repo or manually aggregate results.

**Proposed Solution:**
- Config file specifying multiple repo paths
- Unified drift detection across all repos
- Aggregated reports showing cross-repo inconsistencies

**Estimated Effort:** High  
**User Demand:** Awaiting feedback  
**Breaking Change:** No  

```yaml
# svccat.toml
repos:
  - path: /path/to/repo1
    manifest: services.yaml
  - path: /path/to/repo2
    manifest: infra/manifest.yaml
```

---

#### 2. Custom Validation Rules (Policy as Code)
**Problem:** Teams have different validation policies; one size doesn't fit all.

**Proposed Solution:**
- CEL (Common Expression Language) or Rego-based rule definitions
- Rules for naming conventions, required fields, allowed platforms
- Extensible validator API

**Estimated Effort:** High  
**User Demand:** Awaiting feedback  
**Breaking Change:** No  

```yaml
# validation-rules.yaml
rules:
  - id: naming_convention
    description: "Services must match pattern service-*"
    expression: "service.name.matches('^service-[a-z0-9-]+$')"
    
  - id: required_platform
    description: "All services must specify a platform"
    expression: "service.platform != null"
```

---

#### 3. Webhook Integration & Event Streaming
**Problem:** CI/CD systems need real-time drift notifications, not just manual runs.

**Proposed Solution:**
- Webhook server mode for receiving events
- Push notifications on drift detection
- Integration with webhook event systems (GitHub, GitLab, Slack)

**Estimated Effort:** Medium  
**User Demand:** Awaiting feedback  
**Breaking Change:** No  

```bash
svccat serve --webhook-port 8080
# Receives POST /webhook/drift with git event
```

---

#### 4. Advanced Reporting & Analytics
**Problem:** Historical drift tracking and trend analysis not available.

**Proposed Solution:**
- Drift history database (SQLite or JSON)
- Trend reports (how drift increases over time)
- Service health dashboard
- Export to JSON, CSV, or Prometheus metrics

**Estimated Effort:** Medium  
**User Demand:** Awaiting feedback  
**Breaking Change:** No  

```bash
svccat report --format=prometheus
# Outputs metrics: svccat_drift_count{repo="...",service="..."}
```

---

#### 5. OPA (Open Policy Agent) Integration
**Problem:** Teams using OPA for policy enforcement want to integrate svccat validation with OPA.

**Proposed Solution:**
- OPA bundle support for validation rules
- Rego-based manifest validation
- Compliance reporting

**Estimated Effort:** High  
**User Demand:** Awaiting feedback  
**Breaking Change:** No  

---

#### 6. Service Dependency Graph Visualization
**Problem:** Understanding service dependencies and impact analysis is manual.

**Proposed Solution:**
- Graph visualization of service dependencies
- Impact analysis (which services are affected by a change)
- Export to Mermaid, DOT, or SVG

**Estimated Effort:** Medium  
**User Demand:** Awaiting feedback  
**Breaking Change:** No  

---

### Medium-Value Candidates

#### 7. Template-Based Manifest Generation
- Auto-generate manifest stubs for new services
- Support for Cookiecutters or custom templates

#### 8. Slack/Discord Notifications
- Drift notifications in chat
- Interactive resolution commands

#### 9. Service Maturity Levels
- Track adoption of security practices
- Deprecation tracking for old platforms

#### 10. Batch Operations
- Update multiple services at once
- Bulk validation and reporting
