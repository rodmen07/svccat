## Security Fixes Validation

### ✅ Git Command Injection Prevention

**Test:** Git ref validation prevents command injection

```bash
# Should FAIL (contains command substitution)
svccat check -s 'main; echo hacked' 

# Should FAIL (contains shell metacharacters)
svccat check -s '$(whoami)'

# Should PASS (valid git ref)
svccat check -s 'main'
svccat check -s 'v0.19.0'
svccat check -s 'HEAD~1'
svccat check -s 'feature/my-branch'
```

**Code Location:** `src/since.rs:validate_git_ref()`  
**Test:** `tests/security_tests.rs` + integration tests

---

### ✅ SSRF Prevention (Server-Side Request Forgery)

**Test:** URL validation blocks private IP addresses

```bash
# Create webhook pointing to private IP
# manifest.yaml:
webhooks:
  - url: "http://127.0.0.1:8080/hook"  # Should REJECT

# Create webhook pointing to private range
webhooks:
  - url: "http://10.0.0.5/hook"        # Should REJECT
  - url: "http://192.168.1.1/hook"     # Should REJECT
  - url: "http://[::1]/hook"           # Should REJECT (IPv6 loopback)

# Should ACCEPT (public IP)
webhooks:
  - url: "https://example.com/hook"    # OK
```

**Code Location:** `src/urlvalidation.rs:validate_url()`  
**Test:** `tests/security_tests.rs::test_ipv4_private_rejected`, etc.

---

### ✅ Deserialization Bomb Protection

**Test:** Manifest size limits enforced

```bash
# Create oversized manifest (11 MB)
dd if=/dev/zero of=large-manifest.yaml bs=1M count=11
# Should FAIL with size limit error

# Create manifest with 10,000+ services
# Should FAIL with service count limit
```

**Code Location:** `src/manifest.rs::validate_limits()`  
**Limits:**
- Max file size: 10 MB
- Max services: 10,000
- Max service name length: 256 bytes

**Test:** `tests/security_tests.rs::test_oversized_manifest`, etc.

---

### ✅ Path Traversal Prevention

**Test:** Directory escape attempts blocked

```bash
# manifest.yaml with path traversal
services:
  - name: api
    path: "../../etc/passwd"          # Should REJECT

# Absolute paths
services:
  - name: api
    path: "/etc/passwd"               # Should REJECT

# Windows paths
services:
  - name: api
    path: "C:\\Windows\\System32"     # Should REJECT (all platforms)
```

**Code Location:** `src/manifest.rs:validate_optional_path()`  
**Test:** `tests/security_tests.rs` path validation tests

---

### ✅ Symlink Attack Prevention

**Test:** Symlinks skipped during discovery

```bash
# Create real service
mkdir -p services/real-api
touch services/real-api/Cargo.toml

# Create symlink to external directory
ln -s /etc services/symlink

# Run discovery
svccat check
# Should find: real-api
# Should skip: symlink (with warning)
```

**Code Location:** `src/discovery.rs::discover_services()`  
**Test:** `tests/security_tests.rs::test_skip_symlinks`

---

### ✅ Glob Pattern DoS Prevention

**Test:** Glob pattern limits enforced

```bash
# Too many patterns (limit: 20)
discovery:
  paths:
    - "services00/*"
    - "services01/*"
    - # ... (30 total paths)
# Should warn/reject excessive patterns

# Too many consecutive wildcards (limit: 2)
discovery:
  paths:
    - "services/**/*/src/**/*"
# Should detect and warn
```

**Code Location:** `src/discovery.rs:has_dangerous_pattern()`  
**Test:** `tests/security_tests.rs::test_excessive_glob`

## Known Limitations & Next Steps

**Known Limitations:**
- IPv6 private range detection supports standard private ranges (::1, fe80::/10, fc00::/7)
- Git command injection prevention doesn't support commands; only git refs
- Manifest size limit is fixed (10 MB) and not configurable
- No persistent drift history (point-in-time only)

**Next Steps (v0.20.0):**
1. Gather community feedback on features
2. Prioritize highest-value candidates
3. Implement multi-repo support or policy-as-code
4. Add drift history tracking
5. Enhanced reporting (trends, analytics)
